# 🛍️ متجر Smile Shop

متجر إلكتروني متكامل وعصري للبيع بالتجزئة مصمم بأسلوب بصري فريد وفخم لعام 2026. يتميز بتجربة مستخدم سلسة ونظام إدارة مشرفين آمن وحماية متطورة للمخزون والأداء.

---

## ✨ المميزات الرئيسية

### 🎨 الواجهة وتجربة المستخدم (UX/UI)
- **تصميم فخم وعصري**: خلفيات هلامية متحركة (Aurora Blobs)، حواف زجاجية فخمة (Etched Glass)، وتأثيرات تركيز الضوء (Spotlight).
- **العرض السريع (Quick View)**: إمكانية تصفح تفاصيل المنتج وتحديد الكميات وإضافتها مباشرة للسلة بدون مغادرة الصفحة.
- **منتقي مدن ذكي**: واجهة تفاعلية خالية من حقول الإدخال التقليدية مع بحث وتصفية لحظية لمدن ليبيا وتكلفة الشحن الخاصة بكل مدينة.
- **تتبع الطلبات**: واجهة تتبع فخمة وتفاعلية تدعم التتبع عبر رقم هاتف العميل ورقم الطلب القصير المكون من 6 أرقام.
- **مؤشر الشحن المجاني**: شريط تقدم تفاعلي يوضح للعميل القيمة المتبقية للحصول على شحن مجاني (العتبة الحالية: 150 دينار).

### ⚙️ النظام الخلفي والأمان (Backend & Security)
- **منع حالات السباق (Race Condition)**: تحديث ذري وقوي للمخزون في خطوة واحدة أثناء الطلب، مع آلية تراجع (Rollback) عند فشل أي خطوة لمنع ضياع أو تسرب السلع.
- **حماية أرقام المخزون**: إخفاء الأرقام الدقيقية للمخزون في الواجهات وفي رسائل الخطأ والـ API لحماية بيانات الأعمال الحساسة من المنافسين.
- **نظام المشرفين الآمن (Multi-Admin)**: صلاحيات مقسمة بين المالك (Owner) والمدير (Editor) لحماية البيانات الحساسة مثل حذف الطلبات أو تعديل المشرفين.
- **الأداء العالي**: استخدام الفهارس (Indexes) وتصدير البيانات عبر البث المستمر (Streaming) لتقليل استهلاك الذاكرة وتخفيف العبء على الخادم.

## 🚀 التثبيت والتشغيل المحلي

### 1. المتطلبات الأساسية
- تثبيت **Node.js** (إصدار 18 فما فوق).
- تثبيت وتشغيل قاعدة بيانات **MongoDB** محلياً أو استخدام حساب MongoDB Atlas.

### 2. التثبيت
قم بتثبيت الحزم المطلوبة عبر منفذ الأوامر:
```bash
npm install
```

### 3. إعداد المتغيرات البيئية `.env`
قم بإنشاء ملف `.env` في المجلد الرئيسي للمشروع بناءً على `.env.example`:
```env
PORT=3000
MONGO_URI=mongodb://127.0.0.1:27017/smile_shop
JWT_SECRET=super_secret_key_change_me
ADMIN_USERNAME=your_admin_username
ADMIN_PASSWORD=your_strong_password
```
*ملاحظة: سيقوم النظام بإنشاء حساب المشرف المالك الافتراضي (Owner) تلقائياً عند أول تشغيل بناءً على هذه المتغيرات.*

### 4. التشغيل في بيئة التطوير

لتشغيل الخادم المحلي مع التحديث التلقائي للكود (nodemon):
```bash
npm run dev
```

### 5. فحص واختبار الكود

لتشغيل اختبارات الباك اند والتحقق من صحة الكود:
```bash
npm test
```

## 🛠️ أوامر التطوير (NPM Scripts)

المشروع يحتوي على الأوامر التالية المتاحة للتشغيل:
- `npm start`: تشغيل الخادم في بيئة الإنتاج باستخدام Node.js العادي.
- `npm run dev`: تشغيل الخادم في بيئة التطوير مع nodemon (يستخدم npx تلقائياً بدون الحاجة لتثبيته عالمياً).
- `npm test`: تشغيل جميع اختبارات الوحدة (unit tests) المدمجة بالباك اند.
- `npm run validate`: أمر فحص سريع يؤكد سلامة إعدادات الكود والتحقق التلقائي.

## 🌐 خطوات النشر للإنتاج (Deployment)

عند رفع المتجر لبيئة الإنتاج على خادم خارجي (مثل VPS, Heroku, Render):

1. **تحديث ملف `public/config.js`**:
   قم بتغيير عنوان الـ API ليشير إلى رابط سيرفر الإنتاج الفعلي:
   ```javascript
   window.APP_CONFIG = {
       API_URL: 'https://api.yourdomain.com/api' // أو رابط السيرفر الخاص بك
   };
   ```

2. **تثبيت المتغيرات في الخادم**:
   تأكد من كتابة متغيرات `.env` بشكل صحيح على الخادم، وخصوصاً تفعيل بيئة الإنتاج:
   ```env
   NODE_ENV=production
   ```

3. **تشغيل المشروع**:
   يمكن استخدام مدير العمليات `pm2` لضمان استمرار عمل السيرفر في الخلفية:
   ```bash
   npm start
   ```
